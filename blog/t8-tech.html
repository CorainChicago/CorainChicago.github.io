Web Security

You need to care, at least know to care

September 17, 2015

Web security is an issue involving keeping the website safe, customer information safe, and company data safe, so web security is about safety. When designing a site, security needs to be kept in mind in terms of the code used and the processes enabled to access items.

Normally, systems or devops people focus on security though that doesn't mean the responsibility falls only on their shoulders. It falls on everyone's shoulders to think about it as getting called in on a Saturday night because hackers are stealing customer IDs using your system is no good for anyone.

Here are a few things to always keep in mind (from either a talk I heard or research:

  • Use two factor verification for logins
  • Research before pulling in a gem or open source library (hint, hint, you might be pulling in a security threat)
  • Careful with the information stored in cache or cookies
  • Ruby on Rails Cheatsheet
  • Ruby on Rails Guide to Security

Security is like testing, many devs don't like it until the consequences come. Brush up and embrace it (the Ruby on Rails Guide is the easiest to start reading).
